C++/DirectX11
3D renderer engine

[2021.9~2022.4]

Independent developer


The engine is built on DirectX 11 and Win32 interfaces.


It integrated modern game engine subsystems, including Resource Manager, Job System and Render Graph.


This project emphasized on low coupling and high extensibility. It also comes with full exception handlers for every functional module.


The engine also comes with various shaders that support techniques like Phong Shading, Cel Shading and Percentage Close Soft Shadows. The UI was built with ImGui.



A Cel Shading
Demo Video
(Youtube)

A brief version history is listed below. For complete descriptions, please visit my GitHub repository：


0. 2021-08 Win32 Window, mouse/keyboard IO and basic exception handling.


1. 2021-08 Basic Phong Shading.


2. 2021-09 Drawable (An object containing vertex, index and other model related data) and Bindable (pipeline configurations including shaders and transforms) systems that allowed the system to automatic assembly the rendering process.


3. 2021-10 Bindable Pool system - manage all bindables data to avoid duplicated data. Started to use ImGui.


4. 2021-11 Dynamic Vertex System - Dynamically sets vertex data layout according to the input model data/textures. Register respective vertex data in the Bindable Pool system. Started using AssImp to import models.


5. 2021-12 Dynamic Constant System - sets up the render pipeline and shader combinations according to the vertext layout. The dynamic constants are also registered in the Bindable Pool. Started to use DirectXTex to import textures.


6. 2022-01 Job System and Multi-Pass rendering - registers jobs according to loaded materials and classify them into respective passes.


7. 2022-02 Render Graph System - a Directed Acyclic Graph structure that stores the dependency between passes. Once the materials and pass dependencies are specified, materials can be automatically rendered in correct orders.

Unity2D Game
Jungle Maze

[2022.1~3]

Independent developer/researcher


Jungle Maze is a top-down pixel shooter game, also the research target for my master's thesis. It was used in researching a game mechanic - adaptive procedural content generation. This mechanic focuses on enhancing players' autonomy in the random generation process.


To allow players control the generation, the game drops various supply and map generator cards when enemies are defeated. To collect, use and combine various cards, the future map styles will change accordingly. As most weapons perform differently on different terrains, players need to balance between the weapons and the map generator cards in order to win the game.


Download playable (itch.io)
Read my thesis

Created map generation logic, customized inventory system and weapon system, heavily used event system to keep game codes clean with neat assembly reference seperation.


Designed items and map generation logic.



Click to view
map generation game design

Designed and tuned weapons - by adjusting bullet trace, distribution and firing behaviors, made sure that weapons have balanced but different performances on respective maps.


Designed enemy AI with enemy cards. Enemies are differenciated by weapons. Players thereby need to combine enemy cards with mape generator cards in order to create advantages in the next map


Designed and implemented the card inventory and UI on the top. Optimized interaction experience - game would slow-down when browsing items; same adjacent item cards combine automatically, creating a connect-3 like gameplay; enemy and map cards change periodically, allowing players to exchange waiting time for more possibilities.


The game have both English and Chinese languages

This game was used in 2 research studies:


1. Under the same probability expectation, how could steady compensation (when ammo supply gets low recently, compensate the player with a larger amount of ammo supply) and "super-lucky moment" (record all "bad lucks" and give a "super luck" supply periodically) differ in player experiences?


2. With a consisten art style, how could Adaptive Generation affect player experience compared to normal-crafted game maps? Read the thesis paper

Board Game
Climbers

[2020.7]


An animal battle royale in a forest

  • Designed the initial concept, led several rounds of Brainstorm
  • Designed basic mechanics: turn steps, action points and card system.
  • Designed basic characters and skills
  • Designed terrain system and animal natural traits
  • Instructed team to design animals and tuned the stats

Game Rule:

  • 游戏开始前，所有玩家获得1张角色卡与3张初始卡牌
  • 玩家逆时针进行回合，每回合玩家抽一张卡，然后开始回合
  • 当前玩家投掷一枚硬币，选择本回合的行动点数是奇数(1/3/5)还是偶数(0/2/4)
  • 移动(消耗1)和使用卡牌(消耗因卡而异)都会消耗行动点，玩家必须花完自己声称的行动点
  • 移动会影响玩家在场上的位置，也会影响与其他玩家的距离，攻击等卡牌需要对象在攻击范围内
  • 不同的动物在不同的地形(草原，雨林，沼泽)会有不同的表现，比如，动物的攻击无法穿过自己不善长的地形块(如猴子不能越过沼泽进行攻击)，身陷不擅长地块的动物，攻击距离为1。具体的规则详见游戏指南
  • 当玩家的血量(初始3)减少至0，且没有得到治疗，则玩家被淘汰
  • 回合结束时，玩家手里的卡牌不能超过3张。多出的卡牌需丢弃
  • 每两轮结束时，投掷一枚骰子，对应边的地图将消失，在该地区的动物减少一点血，并强制移动至最近的合法格子
  • 最后留下的玩家获胜

游戏规则

  • 为了保卫南瓜大赛的候选大南瓜，农夫不惜动用田的其他瓜们来对付虫子们
  • 可投掷的瓜在指定区域长大，农夫需要亲自捡起瓜来瞄准，攻击虫子
  • 农夫拥有蓄力条，每当蓄力条满时，农夫可以亲自踢飞一条虫子
  • 当有三只虫子冲破防御，农夫的大南瓜就被咬坏，无法参赛了
